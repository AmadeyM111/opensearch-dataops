Набор утилит для миграции, проверки и тестирования данных в OpenSearch. Проект предназначен для работы с индексами документов.

##  Обзор

Проект предоставляет инструменты для:
- **Миграции данных** между индексами OpenSearch с загрузкой новых данных из CSV
- **Верификации** структуры и качества данных после миграции
- **Тестирования** на малых наборах данных перед полной миграцией
- **Интеграционного тестирования** компонентов системы
- **Поиска** ответов на основе вопросно-ответной системы

##  Структура проекта

```
opensearch-scrutiny/
├── idx-migration.py         # Основной скрипт миграции индекса
├── idx-verify.py            # Проверка результатов миграции
├── test_migration.py        # Тестирование миграции на выборке данных
├── test_integration.py      # Интеграционные тесты компонентов
├── test_agent_answers.py    # Поиск ответов на вопросы
├── sources/                 # Директория для исходных CSV файлов
│   └── Памятка_upsrb_mortgage_memo_20251028.csv
└── .env                     # Переменные окружения (создается пользователем)
```

##  Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repository-url>
   cd opensearch-scrutiny
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```
   Основные зависимости:
   - `opensearch-py`
   - `python-dotenv`
   - `langchain-gigachat` (для векторного поиска)

##  Конфигурация

Создайте файл `.env` в корне проекта со следующими переменными:

```env
# OpenSearch подключение
OPENSEARCH_URL=https://your-opensearch-host
OPENSEARCH_PORT=9200
OPENSEARCH_USER=your-username
OPENSEARCH_PASSWORD=your-password

# GigaChat (для векторных эмбеддингов)
GIGACHAT_SECRET=your-gigachat-secret

# Индексы
MEMO_IDX_NAME=upsrb_mortgage_memo_20251028
```

##  Использование

### 1. Тестирование миграции

Рекомендуется сначала запустить тест на малой выборке данных:

```bash
python test_migration.py
```

Скрипт:
- Создает тестовый индекс
- Загружает 5 документов из CSV
- Проверяет структуру данных
- Удаляет тестовый индекс после проверки

### 2. Полная миграция

После успешного прохождения тестов запустите полную миграцию:

```bash
python idx-migration.py
```

**Что делает скрипт:**
1. Проверяет подключение к OpenSearch
2. Создает новый индекс на основе структуры старого
3. Переливает данные из исходного индекса (reindex)
4. Загружает новые данные из CSV файла
5. Генерирует векторные представления для текстов
6. Проводит валидацию качества данных

**Индексы:**
- Исходный: `upsrb_mortgage_memo_20251028`
- Целевой: `upsrb_mortgage_memo_20260416`

### 3. Проверка результатов

После миграции запустите проверку:

```bash
python idx-verify.py
```

Скрипт проверяет:
- Количество документов
- Структуру данных
- Наличие всех обязательных полей
- Качество векторных представлений

### 4. Интеграционное тестирование

```bash
python test_integration.py
```

Тестирует:
- API клиент Консультант+
- OpenSearch клиент
- Поиск по памятке (текстовый и векторный)
- Полный граф пайплайна

### 5. Поиск ответов на вопросы

```bash
python test_agent_answers.py
```

Осуществляет поиск ответов на вопросы по базе знаний с использованием взвешенного поиска по полям.

##  Скрипты

### idx-migration.py

Основной скрипт для миграции данных между индексами.

**Ключевые функции:**
- `get_embedding()` - генерация векторных представлений (сейчас заглушка)
- `validate_row()` - валидация строк CSV
- `prepare_actions()` - подготовка данных для bulk загрузки

**Важно:** Currently использует детерминированную заглушку для векторов. Замените `get_embedding()` на реальную модель для семантического поиска.

### idx-verify.py

Скрипт для детальной проверки результата миграции.

Проверяет:
- Общее количество документов
- Количество новых/старых документов
- Наличие векторов
- Структуру вложенных объектов (chapter, paragraph, subparagraph)
- Примеры загруженных документов

### test_migration.py

Тестовый скрипт для проверки миграции на малом наборе данных.

**Особенности:**
- Тестирует только первые 5 документов из CSV
- Создает временный индекс `test_migrations_check`
- Проводит валидацию структуры перед загрузкой
- Удаляет тестовый индекс после проверки

**Функции:**
- `validate_document_structure()` - проверка структуры документа
- `prepare_test_documents()` - подготовка тестовых документов

### test_integration.py

Интеграционные тесты для проверки всех компонентов системы.

**Тестируемые компоненты:**
1. API клиент Консультант+ (`test_kplus_client()`)
2. OpenSearch клиент (`test_opensearch_client()`)
3. Поиск по памятке (`test_memo_search()`)
4. Полный граф пайплайна (`test_graph()`)

### test_agent_answers.py

Поиск ответов на вопросы по базе знаний.

**Особенности:**
- Использует взвешенный поиск по полям
- Приоритеты: `conclusion` (5.0) > `clarification` (4.0) > `title` (3.0) > `description` (2.0)
- Форматирует ответ в один абзац
- Ограничивает длину ответа до 800 символов

##  Структура данных

### Документ в индексе

```json
{
  "text": "Полный текст для поиска и векторизации",
  "vector": [массив размерности 2560],
  "chapter": {
    "num": "номер раздела",
    "description": "название раздела"
  },
  "paragraph": {
    "num": "номер пункта",
    "description": "краткое описание",
    "text": "полный текст"
  },
  "subparagraph": {
    "num": "номер подпункта",
    "title": "название подпункта",
    "description": "краткое описание подпункта",
    "clarification": "разъяснения/позиция ЮП",
    "conclusion": "вывод ЮП (ПЭ) в LegalForms",
    "text": "полный текст"
  },
  "insert_date": "ISO дата вставки"
}
```

### CSV структура

Файл `sources/Памятка_upsrb_mortgage_memo_20251028.csv` содержит следующие поля:
- `Номер раздела`
- `Название раздела`
- `Номер пункта`
- `Краткое описание (вопрос)`
- `Номер подпункта`
- `Название подпункта`
- `Краткое описание подпункта`
- `Разъяснения/позиция ЮП`
- `Вывод ЮП (ПЭ) в LegalForms`

- Переменные окружения в `.env` файле
